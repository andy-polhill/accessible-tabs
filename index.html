Accessible Tabs test

The purpose of this page is to offer a few different examples as to how a tab component can be made accessible. The particular concern is around keyboard control and the correct use of aria roles, at present there seems to be two schools of thought as to how a keyboard should be used to control tabs, and it would be good to come up with a general consensus to the best approach.

This is a living document and will be updated with latest feedback and results from user testing.

Tab key controlled Example 1

In this example the user can navigate through the tabs by pressing the tab key, hitting the enter key while the tab is in focus will reveal the content for that tab.
